# USERFLOW: 사용자 및 관리자 여정

## 1. 방문자 흐름 (Visitor Flow)
1. GitHub Pages 주소 접속.
2. 로딩 스피너 확인 (API 데이터 수신 중).
3. 프로젝트 카드 목록 탐색.
4. 특정 프로젝트 카드의 클릭.
5. 새 탭에서 해당 프로젝트 페이지 열림.

## 2. 관리자 흐름 (Admin/Owner Flow)
1. 자신의 GitHub 저장소 접속 -> 'Issues' 탭 이동.
2. 'New Issue' 버튼 클릭.
3. 제목에 프로젝트명 입력.
4. 본문에 첫 줄은 'URL', 다음 줄부터는 '설명' 작성.
5. 'Labels' 섹션에서 `project` 레이블 선택 후 'Submit'.
6. 큐레이션 페이지 접속 시 즉시 업데이트된 리스트 확인.

## 3. 예외 흐름
- **API 호출 실패**: 네트워크 오류 메시지 및 "다시 시도" 버튼 노출.
- **데이터 없음**: 프로젝트 등록 유도 메시지 혹은 빈 화면 처리.

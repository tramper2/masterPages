# PRD: Project Curation Page (GitHub Issues DB)

## 1. 목적
본인이 개발한 여러 프로젝트 웹페이지를 한곳에서 확인하고 이동할 수 있는 "중앙 큐레이션 허브"를 구축한다. 별도의 백엔드 없이 GitHub Issues를 데이터베이스로 활용하여 유지보수 비용을 제로화한다.

## 2. 타겟 사용자
- **방문자**: 개발자의 프로젝트 목록을 확인하고 직접 체험해보고 싶은 사람.
- **관리자(나)**: 새로운 프로젝트가 생길 때마다 복잡한 코드 수정 없이 GitHub Issue 등록만으로 게시물을 추가하고 싶은 개발자.

## 3. 핵심 기능 (MVP)
1. **프로젝트 리스트 렌더링**: GitHub API를 통해 특정 레이블(예: `project`)이 달린 이슈들을 가져와 카드 형태로 표시.
2. **상세 정보 표시**: 이슈의 제목(프로젝트명), 본문(설명), 생성일 등을 표시.
3. **직접 링크 이동**: 각 카드에는 프로젝트로 바로 이동할 수 있는 외부 링크 버튼 포함.
4. **관리자 전용 등록**: GitHub Issue 시스템을 활용하여 관리자만 포스트를 작성/수정/삭제 가능.

## 4. 비기능적 요구사항
- **응답성**: 모바일과 데스크톱 모두에서 잘 보여야 함 (반응형 디자인).
- **성능**: GitHub API 호출 시 로딩 애니메이션 제공.
- **디자인**: 깔끔하고 현대적인 UI (Tailwind CSS 권장).
